# v1 direction

Oxid is still shaping its public surface. The areas below are useful directions, but they should be read as roadmap notes rather than completed features.

## Areas worth expanding

- packaging and installation ergonomics, including a more polished Linux install story
- documentation internationalization built on stable English slugs
- a richer project layout once the scaffold grows beyond the current minimal files
- continued expansion of the native module surface
- better long-term story for editor tooling and typed scripting workflows

## What this page is not

This roadmap is **not** a promise that every item already exists.

For the current shipped flow, rely on:

- `oxid new`
- `oxid run`
- `package.json` with `oxid.entry`
- JavaScript runtime modules
- `oxid.d.ts` for editor support
