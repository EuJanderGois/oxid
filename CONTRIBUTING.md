# Contributing to Oxid

First off, thank you for considering contributing to Oxid! It's people like you that make this game engine better for everyone.

## Getting Started

If you have noticed a bug or have a feature request, please check our **Issues** tab to see if someone else has already reported it. If not, feel free to open a new issue using our templates.

## Development Setup

Oxid is built with a native Rust core and a JavaScript scripting runtime. To start contributing, ensure you have the standard Rust toolchain (Cargo) and your preferred web tooling environment configured on your system.

- Fork the repository
- Clone your fork locally
- Create a specific branch for your task

## Git Workflow and Commit Standards

To maintain a clean and understandable project history, we follow strict guidelines for branching and committing. 

Before making your changes, you **must** read our **[Commit and PR Guidelines](COMMITING.md)**. 

That document contains all the rules regarding:
- Atomic Commits
- Conventional Commits formatting
- Branch naming patterns
- How to properly open and manage a Pull Request
- Post-PR cleanup

## Code Style

- **Rust:** We follow the standard Rust styling. Always run \`cargo fmt\` before staging your files.
- **JavaScript:** Ensure your scripts follow standard modern JavaScript conventions. If a linter or formatter like Prettier is provided in the project, please use it.

We look forward to reviewing your contributions!
